// 01 — ATTACK VECTORS

Common Hacking Techniques

Each technique explained with real mechanics — how attackers execute, what they target, and how to defend.

WEB
SQL Injection
Attackers insert malicious SQL into input fields to manipulate database queries, bypassing authentication or dumping data.
  • Identify unsanitized input fields
  • Inject payload: ' OR 1=1 --
  • Database executes attacker's query
  • Data extracted or auth bypassed
NETWORK
Man-in-the-Middle
Attacker secretly intercepts and potentially alters communications between two parties who believe they're talking directly.
  • ARP spoofing to redirect traffic
  • Position between victim and gateway
  • Intercept & inspect packets
  • Optionally inject malicious content
SOCIAL
Phishing
Deceiving users into revealing credentials or installing malware through fake emails, websites, or messages that appear legitimate.
  • Clone legitimate login page
  • Craft convincing lure email
  • Victim submits credentials
  • Credentials captured by attacker
EXPLOIT
Buffer Overflow
Writing data beyond a buffer's allocated boundary to overwrite adjacent memory — potentially hijacking program execution flow.
  • Locate vulnerable input buffer
  • Craft oversized payload
  • Overwrite return address
  • Redirect execution to shellcode
WEB
Cross-Site Scripting (XSS)
Injecting malicious scripts into trusted websites that execute in other users' browsers, stealing cookies or hijacking sessions.
  • Find unescaped output in app
  • Inject <script> payload
  • Victim's browser executes script
  • Session token or data stolen
NETWORK
Port Scanning & Enumeration
Systematically probing a host's open ports to map running services, operating systems, and potential entry points before an attack.
  • Send SYN packets to port range
  • Analyze RST / SYN-ACK responses
  • Banner-grab service versions
  • Cross-reference known CVEs

// 02 — ATTACK LIFECYCLE

How a Hack Actually Works

Every sophisticated attack follows a structured methodology. Understanding it helps defenders anticipate each phase.

PHASE 01
Reconnaissance
Passive and active information gathering — OSINT on the target, DNS enumeration, employee profiling via LinkedIn, and scanning for exposed infrastructure using Shodan.
PHASE 02
Scanning & Enumeration
Active probing with Nmap, Nessus, or Nikto to discover open ports, service versions, and misconfigured systems. Building a map of the attack surface.
PHASE 03
Gaining Access
Exploiting a vulnerability — whether a CVE in an unpatched service, a weak password through brute force, or a phishing campaign targeting a privileged user.
PHASE 04
Maintaining Persistence
Installing backdoors, adding rogue admin accounts, or deploying rootkits to ensure the attacker can return even after credentials are changed or patches are applied.
PHASE 05
Covering Tracks
Clearing event logs, modifying timestamps, and removing artifacts to make forensic investigation difficult and delay detection of the breach.
